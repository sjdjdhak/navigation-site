🔐 新登录系统测试指南

📋 系统概述

新的登录系统已经实现，主要特点：

  • ✅ 简化的账号密码登录界面
  • ✅ 配置信息存储在GitHub仓库中
  • ✅ 支持多用户权限管理
  • ✅ 自动加载对应的GitHub配置
  • ✅ 安全的密码哈希存储

🧪 测试步骤

1. 准备配置文件

# 复制配置文件 cp .admin/config.example.json .admin/config.json cp .admin/users/admin.example.json .admin/users/admin.json

2. 生成密码哈希

# 为密码 "hello123" 生成哈希 node .admin/generate-password-hash.js hello123

3. 配置GitHub Token

编辑 .admin/users/admin.json，将 "ghp_your_token_here" 替换为实际的GitHub Token。

📁 文件结构

.admin/ ├── README.md # 说明文档 ├── DEPLOYMENT.md # 部署指南 ├── config.example.json # 配置模板 ├── generate-password-hash.js # 密码哈希工具 ├── templates/ │ └── user.template.json # 用户配置模板 └── users/ ├── .gitkeep # 保持目录 └── admin.example.json # 管理员配置模板

🛠 如果遇到问题

  • 登录失败 → 检查密码哈希和用户配置
  • 网络错误 → 确认GitHub仓库配置正确
  • 权限错误 → 验证GitHub Token权限
  • 配置丢失 → 重新复制示例配置文件
